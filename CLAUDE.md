# 프로젝트 가이드

## 블로그 글 작성 규칙

### 문체

- **경어체 사용**: "~입니다", "~합니다" 체를 사용한다
- **직접적인 표현**: 불필요한 수식어를 피하고 핵심을 바로 전달한다
- **독자에게 말하듯이**: "~을 수없이 만났을 것입니다", "~라고 생각할 수 있습니다"

### 글 구조

1. **도입**: 왜 이 주제가 중요한지, 실무에서 어떤 문제와 연결되는지 설명
2. **본문**: 개념 → 동작 원리 → 실제 코드 예시 순서로 전개
3. **정리**: 핵심 내용을 bullet point로 요약

### 설명 방식

- **문제 제시 → 해결책** 패턴 사용
- **비교 표**: Java와 비교, 기술 간 비교 시 표 활용
- **실무 예시**: Spring, DB 등 실제 사용 사례 포함
- **코드**: 개념마다 코드 예시 제공, 주석으로 핵심 포인트 표시

### 메타데이터 (frontmatter)

```yaml
---
title: '제목 - 부제목'
description: '한 줄 설명'
pubDate: 'Jan 01 2025'  # 또는 2025-01-01
tags: ['태그1', '태그2']
series: 'series-name'      # 시리즈인 경우
seriesOrder: 1             # 시리즈 순서
---
```

### 피해야 할 것

- 장황한 서론
- 뻔한 정의로 시작 ("X란 Y입니다"로만 시작하지 않기)
- 근거 없는 "최고", "완벽" 같은 표현
- 이모지 남용
