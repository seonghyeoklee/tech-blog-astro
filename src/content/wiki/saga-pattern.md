# SAGA 패턴

마이크로서비스 환경에서 여러 서비스에 걸친 트랜잭션을 관리하는 패턴입니다.

## 동작 방식

1. 각 서비스는 로컬 트랜잭션 실행
2. 성공하면 다음 서비스 호출
3. 실패하면 이전 단계들의 보상 트랜잭션 실행

## 종류

**Choreography**: 이벤트 기반, 각 서비스가 자율적으로 반응
**Orchestration**: 중앙 오케스트레이터가 흐름 제어

## 예시

```
주문 생성 → 결제 처리 → 재고 차감 → 배송 생성
    ↓ (결제 실패 시)
주문 취소 (보상 트랜잭션)
```
