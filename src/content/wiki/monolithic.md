# 모놀리식 아키텍처

하나의 코드베이스, 하나의 배포 단위, 하나의 데이터베이스를 사용하는 전통적인 아키텍처 방식입니다.

## 특징

- 전체 시스템이 하나의 프로세스로 실행
- 로컬 메서드 호출로 모듈 간 통신
- 단일 데이터베이스로 트랜잭션 관리 용이

## 장점

- 개발, 테스트, 디버깅이 단순
- 로컬 트랜잭션으로 데이터 일관성 보장
- 배포 파이프라인 구성이 간단

## 단점

- 코드 규모 증가 시 빌드/배포 시간 증가
- 일부만 수정해도 전체 재배포 필요
- 전체 시스템을 함께 스케일 아웃해야 함
